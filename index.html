      API request structure of DAH.v1.0.0
      Description

      This is a generated example of the json structure that entities should follow when submitting data to the endpoint of the API. The values are generated according to the validation rules applied to the incoming data. Below you can find both an example of POST request used for submitting a new report, as well as a PUT request to update data in an existing report.

      Notice that this is not the complete API call. You will need to attach an authorization JWT token to the request as well.

      For more in-depth information on the process of submitting data to the API, please visit our Swagger UI page. There you can find an overview of the API endpoints, as well as detailed descriptions of the available API requests that be made to each endpoint.
